# 🦈 Chapter 02: Shark Dentist Challenge

### 🌟 Technical Overview
- **Title:** Shark Dentist Challenge
- **Status:** Fully Integrated
- **Game Type:** `external`
- **External URL:** `/games/shark-dentist/index.html`
- **Primary Component:** `src/components/games/ExternalGameWrapper.tsx`

---

### 🎨 Frontend Details
- **UI Logic:** Loads in an `<iframe>` within the `PhoneFrame`.
- **Config:** 
    - `germSpeed`: 1.5
    - `spawnRate`: 1000ms
- **Communication:** Uses `window.postMessage` to listen for `CHAPTER_COMPLETE` events from the iframe.

---

### 🧠 Core Game Logic (Blueprint: `SharkDentistLogic.gd`)
- **Mechanic:** Precision Target Extraction.
- **Mouth Setup:** 20 distinct "Tooth Spots."
- **Logic:**
    - Spots periodically turn "Rotten" or "Dirty."
    - **Brush Tool:** Reduces "Dirty" status health by 25.
    - **Extractor Tool:** Reduces "Rotten" status health by 50.
- **Win Condition:** Maintain Shark Health above 0 for 60 seconds.

---

### 🔄 Data & Sync Flow
1. **Sync:** `GameContext` debounces the result and sends a `POST` request to `/game/{uid}/chapter-complete`.
2. **Quest:** Checks if it contributes to the "Shark Savior" achievement in `src/services/rpgService.ts`.
3. **Storage:** Local `localStorage` is updated immediately for offline support.
