# 💎 Chapter 01: Cavity Miner Adventure

### 🌟 Technical Overview
- **Title:** Cavity Miner Adventure
- **Status:** Integrated & Playable
- **Game Type:** `external` (Redirected to `plaque-pluck` or `cavity-miner` via `GameEngine`)
- **Primary Component:** `src/components/games/EnamelCastleSiege.tsx`

---

### 🎨 Frontend Details
- **UI Logic:** Grid-based movement via the "Dashboard" and transition to the "Chapter Screen."
- **Visual Styles:** Uses `assetBrushingBasics` for the thumbnail and `toothKingdomBg` for the gameplay background.
- **Controls:** Handles `handleStartChapter(1)` which triggers the `GameEngine`.
- **Key State:** `activeChapterId` is set to `1`.

---

### 🧠 Core Game Logic (Blueprint: `CavityMinerLogic.cs`)
- **Mechanic:** Grid-Exploration & Mining.
- **Grid Size:** 10x10.
- **Player Pos:** Starts at `(5, 5)`.
- **Logic:**
    - Player moves in discrete steps (Up, Down, Left, Right).
    - Each tile has a 15% chance of containing a **Cavity**.
    - Mining a "Cavity" tile increments `cavitiesCollected`.
- **Win Condition:** Clear 10 Cavities or reach 50 moves.

---

### 🔄 Data & Sync Flow
1. **Completion:** `onComplete(score, stars)` is called.
2. **Rewards:** `rpgService.rewardTaskCompletion(userData, 'lesson')` adds **50 XP** and **20 Gold**.
3. **Database:** Updates `character_stats` table in SQLite with the new `xp` and `gold`.
4. **Log:** Backend `main.py` logs the session into the `brushing_logs` if it involves a brushing task.
